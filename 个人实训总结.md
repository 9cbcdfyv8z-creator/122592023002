本次居家实训我独立完成了一套完整的前后端分离图书借阅管理系统，前端基于 Next.js+TypeScript 开发，后端采用 Flask+SQLite 轻量化架构，全程依托 AI 辅助完成编码、接口联调、BUG 修复与全套工程文档编写，完整走完需求设计、功能开发、错误排查、规范交付的全流程开发流程。

开发前期，我主要完成基础页面搭建与后端接口对接，同时解决了大量前端基础问题：项目中大量任意 any 类型导致 TypeScript 不规范警告、useEffect 直接编写异步函数触发 ESLint 语法报错、SSR 服务端渲染环境直接读取 localStorage 导致 window 未定义报错。在 AI 辅助下，我统一采用惰性初始化 Token、IIFE 自执行异步函数、自定义强类型约束，一次性消除项目所有编译警告，统一了全局代码规范。

前后端联调阶段是本次开发的重点难点，项目多次出现跨域拦截问题。取消收藏等接口会被浏览器触发 OPTIONS 预检请求，后端未适配预检路由导致请求直接报错。通过完善全局 CORS 跨域配置，单独兼容所有接口的 OPTIONS 预检请求，彻底解决了项目全部跨域问题，保证删除、编辑、收藏等功能正常使用。

本次开发耗时最长、收获最大的是数据渲染与业务 BUG 排查。借阅审批页面、个人借阅记录、评论管理页面均采用后端多表联查返回元组数组，我最初前端下标取值错乱，导致用户名、图书名称、发布时间全部显示为数字 ID。借助 AI 精准分析后端 SQL 返回字段顺序，逐一对齐前端表格下标，成功修复所有数据错乱问题。同时修复了图书搜索失效、注册邮箱唯一约束冲突、数据库锁定、JSX 语法错误、用户编辑功能缺失等一系列典型问题，让项目功能完整可用、运行稳定。

通过本次实训，我对 AI 辅助开发建立了清晰、客观的认知：AI 能够快速定位底层报错、生成标准化代码、规范接口文档、辅助排查疑难问题，但 AI 代码存在适配漏洞与复制冗余问题，必须依靠人工校验、调试、优化才能落地。同时我熟练掌握了 Git 规范提交、接口文档编写、代码评审、实训报告撰写等工程化交付能力，不再只追求功能跑通，而是兼顾代码规范、异常处理、项目稳定性与交付完整性。

本次独立实训让我完整掌握了前后端分离项目的开发思维与调试思路，综合实践能力得到显著提升。后续我将根据代码评审建议，继续优化环境变量配置、代码精简、异常统一处理、单元测试等工程化内容，进一步提升项目的规范性与完整性。
