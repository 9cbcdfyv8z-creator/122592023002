# 图书借阅管理系统 Next.js + Flask 全量代码评审
## 评审范围
前端：frontend/src/app 全部页面组件
后端：backend/app.py 接口、数据库初始化逻辑
## 一、前端代码优点
1. 全部页面统一添加"use client"，严格遵循Next.js App Router客户端组件规范；
2. 消除全局any类型，自定义BookItem、BorrowItem、UserInfo、CategoryItem强类型，TS无警告；
3. Token统一采用useState惰性初始化，彻底解决SSR环境window、localStorage未定义报错；
4. useEffect异步逻辑全部使用IIFE自执行函数，消除ESLint异步渲染警告；
5. 所有删除、收藏、借阅、搜索接口均增加try-catch异常捕获，区分跨域/业务错误，弹窗友好提示；
6. 图书封面统一配置图片加载失败兜底图标，页面容错性高；
7. 分页、切换分类、切换排序时自动重置页码为1，交互逻辑统一；
8. 收藏、审批、归还操作后自动刷新列表，前端数据实时同步；
9. 修复多层Link嵌套、动态路由跳转失效问题，页面跳转稳定；
10. 搜索逻辑分层：临时输入值与提交值分离，输入文字不会自动刷新页面，体验更佳。

## 二、前端现存缺陷 & 优化方案
1. Axios鉴权请求代码大量重复
优化：封装全局axios拦截器，自动携带token，统一拦截401未登录跳转登录页
2. 仅图书页面有loading，其余页面无加载状态
优化：封装通用Loading全局组件
3. 原生alert弹窗体验简陋
优化封装Toast轻提示组件替代alert
4. 图片错误兜底代码重复冗余
优化封装通用BookCover封面组件复用逻辑
5. 无路由权限守卫，游客可手动输入管理员地址
优化封装高阶鉴权组件区分普通用户/管理员访问权限

## 三、后端代码优点
1. 全局CORS完整配置，放行GET/POST/PUT/DELETE/OPTIONS全部请求与Authorization头，解决浏览器预检跨域；
2. 所有SQL语句使用参数化查询，无字符串拼接，杜绝SQL注入安全漏洞；
3. 完善业务拦截：删除用户前校验关联借阅记录、库存不足禁止借阅、仅借阅中图书可归还；
4. 封装get_token_user、is_admin公共鉴权函数，复用鉴权逻辑；
5. 项目启动自动初始化全套数据表，预置管理员账号与基础图书分类，开箱即用；
6. 兼容前端不规则请求路由（/api/collect/del），前后端路由匹配无冲突；
7. DELETE接口单独处理OPTIONS预检请求，彻底解决删除跨域拦截报错。

## 四、后端缺陷 & 工程化优化建议
1. JWT密钥硬编码在app.py源码中，存在安全泄露风险
优化：新建.env环境变量文件存储SECRET_KEY，使用python-dotenv读取
2. 每个接口重复创建、关闭SQLite数据库连接，IO开销大
优化：封装全局DB工具类复用连接
3. 无统一全局异常捕获，每个接口单独写try-catch
优化：注册Flask全局错误中间件统一返回错误格式
4. 无请求日志、错误日志记录，线上难以排查问题
优化：引入logging模块记录接口访问与报错堆栈
5. 缺少请求参数合法性校验、接口限流机制
6. SQLite仅适合本地演示，生产环境建议切换MySQL

## 五、整体工程化加分优化方向（实训加分项）
1. 前端Jest单元测试、后端PyTest接口自动化测试
2. 前后端统一.env区分开发/生产环境配置
3. GitHub Actions CI自动化代码校验、构建
4. 抽离全局常量、接口地址、状态映射，统一管理
5. 拆分前端接口请求文件，页面只调用函数，不重复写axios代码
