# 图书借阅管理系统 AI 辅助开发交互完整日志
## 说明
本文档记录项目从初始化、页面搭建、接口联调、BUG 排查、功能优化全流程 AI 对话记录，每条日志标注**对应文件、问题现象、AI 提供方案、修复结果**，用于实训考核 AI 工具运用校验。

## 1 项目初始化与基础页面搭建
### 日志1
对应文件：全部前端 page.tsx、后端 app.py
用户需求：基于 Next.js App Router + Flask 搭建图书借阅前后端项目，实现多页面布局、分页列表、分类筛选，消除 TS any 警告、SSR window 报错、useEffect 异步 ESLint 警告。
问题现象：
1. 页面大量使用 any 模糊类型，TS 持续告警；
2. useEffect 内部直接书写 async 函数，ESLint 报错；
3. useEffect 内直接读取 localStorage，服务端渲染 window 未定义；
AI 解决方案：
1. 自定义 BookItem、CategoryItem、BorrowItem、UserInfo 强类型，替换所有 any；
2. useEffect 内部使用 `(async ()=>{})()` IIFE 自执行函数处理异步请求；
3. token 采用 `useState(()=>{})` 惰性初始化，仅客户端读取存储，不在 Effect 操作 localStorage；
修复结果：页面无 TS 警告、无 SSR 渲染报错、ESLint 规则全部通过，基础图书列表、分页、分类功能正常。

### 日志2
对应文件：src/app/book/page.tsx、后端 /api/book/list
用户需求：实现前台图书页面布局、树形分类渲染、多条件参数拼接、多维度排序。
问题现象：页面布局杂乱，请求参数拼接繁琐，分类树形结构渲染逻辑复杂。
AI 解决方案：
1. 左右分栏布局，左侧树形分类、右侧图书网格卡片；
2. 使用 URLSearchParams 统一管理 page、size、category_id、sort、keyword 请求参数；
3. 递归函数解析后端分类树数据，实现父子层级展示；
修复结果：前台图书基础浏览、分类切换、排序切换功能完整可用。

## 2 早期跨域、收藏、路由类BUG修复
### 日志3
对应文件：backend/app.py、src/app/book/page.tsx
用户需求：解决 DELETE 收藏接口 CORS 预检报错，收藏/取消收藏状态不刷新。
问题现象：浏览器发送 DELETE 请求自动触发 OPTIONS 预检，后端无对应处理逻辑，跨域拦截；操作后收藏按钮状态不会同步更新。
AI 解决方案：
1. 后端全局 CORS 配置放行 GET/POST/PUT/DELETE/OPTIONS，开放 Authorization 请求头；
2. 后端新增 `/api/collect/del/<bid>` 兼容路由，匹配前端请求地址；
3. 收藏操作完成后重新调用接口拉取最新收藏 ID 列表；
修复结果：收藏、取消收藏无跨域报错，切换页面/操作后按钮状态实时同步。

### 日志4
对应文件：src/app/book/page.tsx、src/app/profile/page.tsx
用户需求：修复图书卡片多层 Link 嵌套，封面区域无法跳转详情页。
问题现象：卡片内部嵌套多层 `<Link>` 标签，点击无跳转，仅文字可点，封面图片无响应。
AI 解决方案：移除嵌套 Link，外层统一包裹跳转区域，详情按钮使用 useRouter 编程式跳转；
修复结果：图书卡片任意区域均可点击进入详情动态路由。

### 日志5
对应文件：src/app/profile/page.tsx、后端 /api/borrow/return/<borrow_id>
用户需求：归还图书接口 405 Method Not Allowed 报错。
问题现象：前端使用 POST 请求调用归还接口，后端仅支持 PUT 请求方式。
AI 解决方案：修改 axios 请求 method 为 put，携带登录鉴权 token；
修复结果：用户正常归还图书，后端同步更新图书库存、借阅状态。

### 日志6
对应文件：src/app/admin/borrow/page.tsx、后端 /api/admin/borrow/list
用户需求：管理员借阅页面增加筛选功能，数字状态转中文文字展示。
问题现象：无逾期、即将到期筛选条件，状态数字直接展示，可读性差。
AI 解决方案：
1. 后端接口增加 filter 参数区分 all/overdue/soon；
2. 前端定义 statusMap 对象，将 0/1/2/3/4 转为待审批、借阅中等中文；
修复结果：管理员可多条件筛选全部借阅记录，表格状态清晰易懂。

## 3 管理员后台核心BUG批量修复（本次重点修复项）
### 日志7
对应文件：src/app/admin/user/page.tsx、后端 /api/admin/user/del/<uid>
用户需求：管理员用户编辑、删除功能失效，DELETE 请求 OPTIONS 预检跨域拦截，存在借阅用户无法删除无提示。
问题现象：
1. 编辑按钮仅弹窗提示文字，无表单、无 PUT 修改接口逻辑；
2. 删除用户时，若用户存在关联借阅数据，后端报错前端无友好提示；
3. 删除 DELETE 请求触发 OPTIONS，后端未单独处理预检请求，跨域拦截无法删除；
AI 解决方案：
1. 新增编辑弹窗、表单回填逻辑，封装 PUT 修改用户接口；
2. 删除增加 try-catch 捕获后端业务提示，区分网络错误与业务限制；
3. 后端删除路由单独监听 OPTIONS 请求，直接返回 200 状态码，消除预检跨域；
修复结果：管理员可完整编辑用户昵称、角色、账号状态；存在借阅数据弹窗提示无法删除，无跨域拦截。

### 日志8
对应文件：src/app/admin/borrow/page.tsx、src/app/profile/page.tsx
用户需求：借阅审批列表、个人借阅列表图书名称、用户名全部显示数字ID，无法展示文本。
问题根源：后端多表联查 SQL `br.*,u.username,b.name` 返回元组数组，前端数组下标取值错误。
AI 解决方案：
1. 借阅审批页面：用户名 item[2] → item[10]，图书名称 item[4] → item[11]；
2. 个人中心借阅表格：图书名称 row[4] → row[10]；
修复结果：表格正常展示用户名、完整图书名称，不再出现纯数字。

### 日志9
对应文件：src/app/admin/borrow/page.tsx
用户需求：修复 JSX 编译报错 `Expected '</', got 'ident'`。
问题现象：复制粘贴代码产生冗余重复字符串，JSX td 标签语法断裂，项目构建失败。
AI 解决方案：定位报错行，删除多余重复文本，修正完整标签结构；
修复结果：页面无构建错误，可正常编译渲染表格。

### 日志10
对应文件：src/app/book/page.tsx、后端 /api/book/list
用户需求：前端搜索传参正常，但图书列表不会根据关键词过滤，搜索功能失效。
问题根源：
1. 后端接口未拼接 keyword 模糊查询 WHERE 条件，接收参数但不做过滤；
2. 前端输入文字实时触发请求，搜索按钮逻辑混乱；
AI 解决方案：
1. 后端改造 SQL 逻辑，新增 name、author 双字段模糊匹配，使用参数化查询防 SQL 注入；
2. 前端拆分 searchKey（输入临时值）、finalSearch（提交生效值），仅点击搜索/回车触发接口，切换分类、分页自动重置页码，新增 loading 状态防止重复点击；
修复结果：输入书名、作者可精准过滤图书，搜索功能完全正常。
